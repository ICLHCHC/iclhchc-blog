# 接下来是什么？

恭喜您！您现在已经理解了 Docusaurus 的大部分核心功能。您已经：

- [使用页面插件](./creating-pages.mdx) 创建了一个独立的 React / Markdown 页面
- [使用文档插件](./docs/docs-introduction.mdx) 创建了文档页面，包括 [配置侧边栏](./docs/sidebar/index.mdx) 和 [版本管理](./docs/versioning.mdx)
- [使用博客插件](../blog.mdx) 创建了一个功能齐全的博客
- 体验了 [一系列 Markdown 特性](./markdown-features/markdown-features-intro.mdx)，这些特性对所有内容插件都很有用
- [使用样式表](./styling-layout.mdx) 或 [自定义组件](./swizzling.mdx) 来定制您网站的外观
- [在页面中包含图片和其他资产](./static-assets.mdx)
- [为您的网站添加搜索功能](./search.mdx)
- 理解了如何通过标准 Docusaurus API 实现 [浏览器支持](./browser-support.mdx) 和 [SEO](./seo.mdx)
- 学习了如何安装和配置 [单独插件](./using-plugins.mdx)
- [将网站部署](./deployment.mdx) 到内容主机
- [国际化](./i18n/i18n-tutorial.mdx) 您的网站以支持多种语言

此时，您可能已经有了一个较大的 `docusaurus.config.js` 文件😄 但是，您还没有写太多代码！大部分功能都是通过调用封装的 Docusaurus API 实现的。随着您继续前行，您可以选择以下三条路径：

- 学习更高级的 Docusaurus 概念。这将帮助您更深入地理解这些 API 的功能。
- 阅读 [所有现有的 API](../docusaurus-core.mdx)。许多内容在指南中尚未覆盖！
- 学习 [开发插件](../api/plugin-methods/README.mdx) 来扩展您网站的功能。
